# Contributing to Awesome DevSecOps

First off, thank you for taking the time to contribute! It is contributions like yours that make this list a valuable resource for the DevSecOps community.

When contributing, please ensure you follow these guidelines:

## Standards
- **Quality over Quantity**: We only accept tools and resources that are actively maintained and provide real value.
- **Categorization**: Place your suggestion in the most appropriate existing category.
- **Description**: Provide a clear, concise description (usually one sentence) for each entry.
- **Formatting**: Use the format: `[Name](URL) - Short description.`
- **Check for Duplicates**: Search the list to ensure the resource hasn't been added already.

## How to Contribute
1. **Fork** the repository to your own GitHub account.
2. **Create a new branch** for your changes: `git checkout -b add-resource-name`.
3. **Make your changes** in the `README.md` file.
4. **Run a link check**: Ensure your link is working and not leading to a 404 page.
5. **Commit your changes**: Use a descriptive message like `feat: add [Tool Name] to [Category]`.
6. **Push to the branch**: `git push origin add-resource-name`.
7. **Open a Pull Request** and describe why this resource should be included.

## Code of Conduct
By contributing to this project, you agree to abide by the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org).
